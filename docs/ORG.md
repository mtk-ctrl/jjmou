# プログラミングランド 組織体制

```
┌────────────────────────────────────────────────────────┐
│                    👑 OWNER（オーナー）                  │
│              mtk-ctrl / プロジェクト最終意思決定者        │
│              ★ CEOとのみ直接やり取り                      │
└───────────────────────┬────────────────────────────────┘
                        │ 指示・承認
                        ▼
┌────────────────────────────────────────────────────────┐
│              🤖 CEO / REP（Replitエージェント）           │
│                  コードネーム: CEOレップ                  │
│                                                        │
│  役割:                                                  │
│  ・アーキテクチャ設計・実装                               │
│  ・GitHub / Vercel / Supabase 連携                      │
│  ・組織のタスク管理・進捗報告                             │
│  ・リンへのタスク委託・レポート受理                        │
│  ・オーナーへの意思決定エスカレーション                    │
└────────────────┬───────────────────────────────────────┘
                 │ UXレビュー委託
                 ▼
┌────────────────────────────────────────────────────────┐
│               🌸 LIN / リン（UXレビュアーAI）            │
│                                                        │
│  役割:                                                  │
│  ・子供目線（5〜8歳）での全アプリレビュー                  │
│  ・「難しすぎる」「飽きる」「わからない」の洗い出し          │
│  ・改善提案のレポート作成（忖度なし）                      │
│  ・各リリース前のUX品質チェック                           │
│                                                        │
│  ポリシー:                                              │
│  ・子供が楽しめることのみを最優先                          │
│  ・CEOの実装に対しても遠慮なく批判する                     │
│  ・改善提案は具体的かつ実装可能なものにする                 │
└────────────────────────────────────────────────────────┘
```

## 意思決定ルール

| 内容 | 担当 |
|------|------|
| 新機能の方向性 | オーナー指示 → CEO提案 → オーナー承認 |
| 実装・技術決定 | CEO独自判断（オーナー確認不要） |
| UX品質基準 | リン評価 → CEO反映 |
| リリース判断 | CEO → オーナー報告 |

## コミュニケーションチャネル

- **オーナー → CEO**: Replitチャット（本スレッド）
- **CEO → リン**: subagentタスク委託
- **リン → CEO**: `docs/lin-reports/*.md` にレポート提出
- **CEO → オーナー**: チャットで要約報告
