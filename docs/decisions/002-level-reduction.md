# ADR 002: レベルあたりの問題数を10問→5問に削減

- **日付**: 2026-05-06
- **決定者**: オーナー指示 → CEO実装
- **ステータス**: 採用済み

## 背景
オーナーより「1レベル10問は多くて子供が飽きちゃう」との指摘。

## 問題分析
- コードにゃんこ: 10レベル × 10問 = 100問（長すぎる）
- 子供の集中力は1ゲームセッションで20〜30分が限界
- 1レベルのクリアに達成感を感じるには5問が適切

## 決定事項
**全アプリの問題数を5問/レベルに統一する**

| アプリ | 変更前 | 変更後 |
|--------|--------|--------|
| コードにゃんこ | 10レベル × 10問 = 100問 | 10レベル × 5問 = 50問 |
| おんがくプログラミング | 10レベル × 5問 = 50問 ✓ | 変更なし |
| ドット絵プログラミング | 10レベル × 5問 = 50問 ✓ | 変更なし |
| ひらがなかきかた | 10レベル × 5問 = 50問 ✓ | 変更なし |
| かたちパズル | 10レベル × 5問 = 50問 ✓ | 変更なし |

## 実装方法
コードにゃんこの既存 `levels.ts`（100問）をそのまま保持し、
App.tsx と LevelSelect.tsx で `questionNum <= 5` でフィルタリング。
→ データを削除せず、将来的に「ボーナス問題」として復活可能。

## 期待効果
- レベルクリアの達成感が早く得られる（集中力に合った長さ）
- 次のレベルへの期待感が続く
- 全クリアへの心理的ハードルが下がる
