# ADR 001: 初期アーキテクチャ決定

- **日付**: 2026-05-06
- **決定者**: CEO/Rep
- **ステータス**: 採用済み

## 背景
プログラミングランドを単一のコードにゃんこアプリから5アプリスイートに拡張。
複数の子供が同じデバイスで遊べるユーザー管理が必要になった。

## 決定事項

### 1. フロントエンドのみ構成（初期）
**決定**: React + TypeScript + Vite、バックエンドなし（localStorage）  
**理由**: 子供向けゲームは接続不要で動くことが重要。ネット環境がない場所でも遊べる。  
**トレードオフ**: デバイスをまたいだ進捗共有ができない → Supabase連携（v3.1）で解決予定

### 2. 4スロット固定式ユーザー管理
**決定**: メール・パスワードなしの4スロット方式（🐱🐶🐸🐰）  
**理由**: 子供はパスワードを覚えられない。視覚的などうぶつアイコンで即座に識別できる。  
**トレードオフ**: 5人以上は使えない → ご家庭想定では最大4人で十分

### 3. ベースパス /jjmou/
**決定**: `vite.config.ts` の `base: '/jjmou/'` で設定  
**理由**: GitHub Pagesおよびサブパスでのホスティング対応  

### 4. localStorage → Supabase移行戦略
**決定**: localStorageを第一ストレージとし、Supabaseをバックアップ同期として使用  
**理由**: オフライン動作を保証しつつ、デバイス間同期も実現

### 5. リンUXレビュー制度
**決定**: 各メジャーリリース前に子供目線AIレビューを実施  
**理由**: 大人目線の開発では子供が楽しめない機能が混入しがち。外部視点が必要。
